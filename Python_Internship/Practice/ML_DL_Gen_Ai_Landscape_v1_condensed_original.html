Linear Algebra

The language ML is written in. Every neural network layer is a matrix operation. Every gradient is a vector. Understanding this fluently — not just mechanically — is what separates people who use libraries from people who design architectures.

Vectors & Matrices

Vectors

A vector is an ordered list of numbers representing a point (or direction) in multi-dimensional space. In ML, a vector can represent: a single training example (features), a word embedding, the output of a layer, or the gradient of a loss.

x = [x₁, x₂, ..., xₙ]ᵀ (column vector, shape n×1) ‖x‖₂ = √(x₁² + x₂² + ... + xₙ²) (L2 norm — Euclidean length) x · y = Σᵢ xᵢyᵢ = ‖x‖‖y‖cos(θ) (dot product — measures alignment) Vector fundamentals

The dot product tells you how aligned two vectors are. cos(θ) = 1: identical direction. cos(θ) = 0: perpendicular (orthogonal). cos(θ) = -1: opposite. This is exactly how cosine similarity works in recommendation systems and vector databases.

Matrices & Matrix Multiplication

A matrix is a 2D array. The core operation: multiply matrix W (shape m×n) by vector x (shape n×1) → output y (shape m×1). This is how every dense neural network layer works: transform data from one representation space to another.

Matrix–Vector Multiplication: y = Wx + b (the neural layer equation)

Singular Value Decomposition (SVD)

SVD is one of the most important matrix operations in ML. Any matrix A (m×n) can be decomposed as: A = UΣVᵀ where U (m×m) and V (n×n) are orthogonal matrices, and Σ (m×n) is diagonal with non-negative singular values σ₁ ≥ σ₂ ≥ ... ≥ 0.

💡 SVD Use Cases

PCA: Principal components = columns of V, variances = σᵢ². Project data to top-k dimensions. LoRA: Fine-tune by approximating weight updates as low-rank matrices ΔW ≈ AB. A and B are much smaller than ΔW — motivated by SVD showing most weight matrices are low-rank. Recommendation systems: Matrix factorization for collaborative filtering.

Calculus for ML

Gradients, Jacobians & Hessians

The gradient of a scalar function f(x₁, x₂, ..., xₙ) is the vector of all partial derivatives. It points in the direction of steepest ascent. We minimize loss by moving opposite to the gradient.

∇f(x) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]ᵀ (gradient — shape same as x) Jacobian J ∈ ℝᵐˣⁿ : Jᵢⱼ = ∂fᵢ/∂xⱼ (for vector-valued f: ℝⁿ → ℝᵐ) Hessian H ∈ ℝⁿˣⁿ : Hᵢⱼ = ∂²f/∂xᵢ∂xⱼ (second-order info — curvature) The hierarchy of derivatives in ML

In practice: gradient = first derivative (used every training step). Jacobian = matrix of first-order partials (used in backprop through layers). Hessian = second-order info (too expensive to compute for large models; approximated in second-order optimizers like K-FAC).

Computational Graphs

A computational graph represents a computation as a directed acyclic graph (DAG) where nodes are operations and edges are tensors. PyTorch builds this graph dynamically during the forward pass and uses it for automatic differentiation (autograd) during the backward pass.

Computational Graph: L = (wx + b - y)² — forward (blue) and backward (orange)
💡 Autograd = Automatic Differentiation

PyTorch's autograd engine records operations during the forward pass (building the computational graph), then traverses it backward applying the chain rule automatically. You never compute gradients by hand in practice — but knowing how they're computed is essential for debugging and designing new architectures.

Maximum Likelihood, Cross-Entropy & KL Divergence

Maximum Likelihood Estimation (MLE): find model parameters θ that maximize the probability of observed data. Training a neural network is MLE — minimizing cross-entropy loss is equivalent to maximizing log-likelihood of the data under the model.

MLE: θ* = argmax_θ Σᵢ log p(yᵢ | xᵢ; θ) Negative log-likelihood = Cross-Entropy loss: L = -Σᵢ Σₖ yᵢₖ · log(p̂ᵢₖ) (for K classes, one-hot yᵢₖ) KL Divergence: D_KL(P‖Q) = Σₓ P(x) · log(P(x)/Q(x)) = H(P,Q) - H(P) (Cross-entropy minus entropy) Key information-theoretic relationships
📌 Why Cross-Entropy as Loss?

Cross-entropy H(p,q) = -Σ p(x) log q(x) measures "how many bits are needed to represent events from distribution p using code optimized for q." When p = true labels (one-hot) and q = model predictions, minimizing cross-entropy = maximizing likelihood = the model learning to assign high probability to correct answers. This is the MLE framing of classification training.

Entropy: H(P) = -Σ P(x) log P(x) — average information content / uncertainty of a distribution. Uniform distribution = maximum entropy. Peaked distribution = low entropy.

KL Divergence: Always ≥ 0. Equals 0 iff P=Q. Not symmetric: D_KL(P‖Q) ≠ D_KL(Q‖P). Used in VAE loss to regularize latent distributions toward N(0,I).

Bias–Variance Tradeoff

Every ML model's generalization error decomposes into three terms:

Expected Error = Bias² + Variance + Irreducible Noise Bias: Error from wrong assumptions. Underfitting. High bias → misses patterns. Variance: Error from sensitivity to training data fluctuations. Overfitting. Noise: Irreducible error inherent in the data (labeling error, fundamental stochasticity). The fundamental decomposition

High bias (underfit): model too simple. Train loss = Val loss = both high. Fix: more parameters, better features, less regularization. High variance (overfit): model too complex. Train loss low, Val loss high. Fix: more data, regularization, dropout, early stopping, smaller model.

Perceptrons & Multi-Layer Networks

The Artificial Neuron

Single Artificial Neuron — Weighted Sum + Activation

Multi-Layer Perceptron (MLP)

MLP: Input (3) → Hidden (4) → Hidden (4) → Output (2)

Each layer computes: h = f(Wh_prev + b) where f is a non-linear activation. Without the non-linearity, any stack of linear layers collapses to a single linear transformation — you can only learn linear functions.

Universal Approximation Theorem

An MLP with one hidden layer and enough neurons can approximate any continuous function on a bounded domain to any desired precision. This is why MLPs are theoretically universal — but in practice, depth (multiple layers) is far more efficient than width (very wide single layer). Deep networks learn hierarchical representations that are more parameter-efficient.

Backpropagation — Full Derivation

Backpropagation is the algorithm computing ∂L/∂w for every weight w in the network, using the chain rule. It's not a separate "thing" — it's automatic differentiation applied to the computational graph.

Step-by-Step Derivation

Consider a 2-layer network for binary classification:

Forward pass: z¹ = W¹x + b¹ (pre-activation layer 1) a¹ = ReLU(z¹) (post-activation layer 1) z² = W²a¹ + b² (pre-activation layer 2) ŷ = σ(z²) = 1/(1+e^{-z²}) (sigmoid for binary) L = -y·log(ŷ) - (1-y)·log(1-ŷ) (binary cross-entropy) Forward pass equations
Backward pass (chain rule, right to left): ∂L/∂ŷ = -y/ŷ + (1-y)/(1-ŷ) (loss w.r.t output) ∂L/∂z² = ŷ - y (sigmoid + CE simplifies beautifully!) ∂L/∂W² = ∂L/∂z² · (a¹)ᵀ (outer product) ∂L/∂b² = ∂L/∂z² ∂L/∂a¹ = (W²)ᵀ · ∂L/∂z² (propagate error back through W²) ∂L/∂z¹ = ∂L/∂a¹ ⊙ ReLU'(z¹) (⊙ = element-wise; ReLU' = 1 if z>0 else 0) ∂L/∂W¹ = ∂L/∂z¹ · xᵀ ∂L/∂b¹ = ∂L/∂z¹ Backward pass — chain rule applied layer by layer

import numpy as np

class TwoLayerNet:
    def __init__(self, n_in, n_hidden, n_out, lr=0.01):
        # Xavier initialization: scale by sqrt(2/n_in)
        self.W1 = np.random.randn(n_hidden, n_in) * np.sqrt(2.0/n_in)
        self.b1 = np.zeros((n_hidden, 1))
        self.W2 = np.random.randn(n_out, n_hidden) * np.sqrt(2.0/n_hidden)
        self.b2 = np.zeros((n_out, 1))
        self.lr = lr

    def relu(self, z): return np.maximum(0, z)
    def relu_grad(self, z): return (z > 0).astype(float)
    def sigmoid(self, z): return 1 / (1 + np.exp(-z))

    def forward(self, x):
        self.x = x  # cache for backprop
        self.z1 = self.W1 @ x + self.b1     # (n_h, batch)
        self.a1 = self.relu(self.z1)         # (n_h, batch)
        self.z2 = self.W2 @ self.a1 + self.b2  # (n_out, batch)
        self.yhat = self.sigmoid(self.z2)    # (n_out, batch)
        return self.yhat

    def backward(self, y):
        m = y.shape[1]  # batch size
        # Gradient of binary cross-entropy + sigmoid: dL/dz2 = ŷ - y
        dz2 = (self.yhat - y) / m          # (n_out, batch)
        dW2 = dz2 @ self.a1.T              # (n_out, n_h)
        db2 = dz2.sum(axis=1, keepdims=True)
        da1 = self.W2.T @ dz2              # (n_h, batch)
        dz1 = da1 * self.relu_grad(self.z1)  # element-wise: ReLU derivative
        dW1 = dz1 @ self.x.T              # (n_h, n_in)
        db1 = dz1.sum(axis=1, keepdims=True)
        # Gradient descent update
        self.W2 -= self.lr * dW2
        self.b2 -= self.lr * db2
        self.W1 -= self.lr * dW1
        self.b1 -= self.lr * db1
📌 The Beautiful Simplification

For sigmoid + binary cross-entropy: ∂L/∂z = ŷ - y. For softmax + categorical cross-entropy: ∂L/∂z = ŷ - y_onehot. These elegant results are why these pairings are universally used. The gradient is simply "prediction minus truth" — intuitive and numerically stable.

Residual Networks — Deep Dive

The key insight of ResNets: instead of learning H(x) directly, learn the residual F(x) = H(x) - x, so the total output is H(x) = F(x) + x. The identity path (skip connection) allows gradients to flow directly backward through layers without multiplying through weight matrices.

Standard block: x_{l+1} = f(W_l × x_l + b_l) Residual block: x_{l+1} = f(F(x_l) + x_l) where F = two conv layers = f(W₂·relu(W₁·x) + x) Gradient: ∂L/∂x_l = ∂L/∂x_{l+1} · (1 + ∂F/∂x_l) The "+1" term ensures gradient ≥ 1 even if ∂F/∂x = 0 → Gradient can flow unchanged through the identity shortcut Why skip connections solve vanishing gradients for depth 
# ResNet block in PyTorch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(channels, channels, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(channels),
            nn.ReLU(inplace=True),
            nn.Conv2d(channels, channels, kernel_size=3, padding=1, bias=False),
            nn.BatchNorm2d(channels)
        )
        self.relu = nn.ReLU(inplace=True)

    def forward(self, x):
        residual = x
        out = self.block(x)
        out = out + residual  # ← the skip connection
        return self.relu(out)

Recurrent Neural Networks

RNN Formulation

At each time step t: hₜ = tanh(Wₓₓ · xₜ + Wₕₕ · hₜ₋₁ + b) (hidden state update) yₜ = Wₕᵧ · hₜ + bᵧ (output if needed at each step) Parameters: Wₓₓ ∈ ℝ^{d_h × d_x}, Wₕₕ ∈ ℝ^{d_h × d_h} Total params ≈ d_h×(d_x + d_h + 1) — same params reused at every step RNN — same weights at every time step (parameter sharing across time)

BPTT & The Vanishing Gradient Problem

Backpropagation Through Time (BPTT): unroll the RNN for T steps, creating a T-layer "network" in time, then apply standard backprop. The gradient of the loss at time T with respect to h₀ requires multiplying the Jacobian of h with respect to itself T times.

∂L/∂h₀ = ∂L/∂hₜ · ∏_{k=1}^{T} ∂hₖ/∂hₖ₋₁ = ∂L/∂hₜ · ∏_{k=1}^{T} (Wₕₕᵀ · diag(1-tanh²(hₖ))) If spectral radius ρ(Wₕₕ) < 1: gradient exponentially → 0 (vanishing) If ρ(Wₕₕ) > 1: gradient exponentially → ∞ (exploding) Why vanilla RNNs can't learn long-range dependencies
⚠️ The Vanishing Gradient Problem — Intuition

Imagine whispering a message down a line of 100 people. Each person adds a small random distortion. By the end, the original message is unrecognizable. The RNN gradient is that message — it gets multiplied by the same weight matrix at each of the T steps. If the weights are slightly less than 1 (which is typical after training), T multiplications = the gradient exponentially shrinks. The model can't adjust weights in early layers because their gradients are essentially zero.

The LSTM Cell

LSTM Cell — Cell state (memory highway) + 3 gates controlling information flow

Gate Equations — Complete Derivation

Input concatenation: [hₜ₋₁, xₜ] ∈ ℝ^{d_h + d_x} Forget gate: fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf) (∈ (0,1)^{d_h}) Input gate: iₜ = σ(Wi · [hₜ₋₁, xₜ] + bi) (∈ (0,1)^{d_h}) Candidate: C̃ₜ = tanh(Wc · [hₜ₋₁, xₜ] + bc) (∈ (-1,1)^{d_h}) Output gate: oₜ = σ(Wo · [hₜ₋₁, xₜ] + bo) (∈ (0,1)^{d_h}) Cell state: Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ ↑ keep old ↑ add new Hidden state: hₜ = oₜ ⊙ tanh(Cₜ) Parameters per gate: (d_h + d_x) × d_h + d_h Total: 4 × [(d_h + d_x) × d_h + d_h] ← 4× more than RNN Complete LSTM equations
💡 Why LSTM Solves Vanishing Gradients

The cell state update is additive: Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ. When backpropagating, the gradient of L with respect to Cₜ₋₁ is ∂L/∂Cₜ · fₜ. If the forget gate learns to output ≈1 (keep everything), the gradient flows back unchanged — no multiplication that causes exponential decay. The cell state is a "highway" for gradients.

Chapter 12

GRU & Bidirectional RNNs

GRU — Gated Recurrent Unit

Reset gate: rₜ = σ(Wr · [hₜ₋₁, xₜ] + br) (how much of past to forget) Update gate: zₜ = σ(Wz · [hₜ₋₁, xₜ] + bz) (balance old vs new) Candidate: h̃ₜ = tanh(W · [rₜ⊙hₜ₋₁, xₜ]) (new candidate, filtered by reset) New state: hₜ = (1-zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ (interpolate old and candidate) GRU — 2 gates instead of LSTM's 3, no separate cell state

GRU merges the forget and input gates into a single "update gate." No separate cell state — the hidden state serves as both. ~25% fewer parameters than LSTM with similar performance. Faster to train. Often preferred on smaller datasets.

Bidirectional RNNs

Process the sequence in both directions. Forward RNN reads left-to-right, backward RNN reads right-to-left. Concatenate (or add) their hidden states at each position. The output at position t sees both past and future context.

Use Cases

Bidirectional LSTM/GRU: Named entity recognition, POS tagging, sentiment analysis — tasks where both context directions matter. BERT is essentially bidirectional Transformer encoding — it reads the full sequence before computing any representation. GPT is unidirectional (causal) — can only look left (needed for generation).

Bahdanau Attention (Additive)

Problem: Encoder compresses entire input into one vector c — information bottleneck Bahdanau solution: compute dynamic context cₜ at each decoder step eₜᵢ = vₐᵀ · tanh(Wₐ·sₜ + Uₐ·hᵢ) (alignment score: decoder state sₜ vs encoder hᵢ) αₜᵢ = softmax(eₜᵢ) (attention weights over all encoder positions) cₜ = Σᵢ αₜᵢ · hᵢ (context vector: weighted sum of encoder states) sₜ = f(sₜ₋₁, yₜ₋₁, cₜ) (decoder state uses dynamic context) Parameters: Wₐ (decoder), Uₐ (encoder), vₐ (scoring vector) — all learned Bahdanau (2015) — first attention mechanism in NLP

Self-Attention — Full Derivation

Query, Key, Value — The Core Idea

Self-attention lets each position in a sequence attend to every other position in the same sequence. Every input element is transformed into three vectors through learned linear projections:

For each input xᵢ ∈ ℝ^d: qᵢ = Wᵠ · xᵢ (Query: "what am I looking for?") kᵢ = Wᴷ · xᵢ (Key: "what do I contain?") vᵢ = Wᵛ · xᵢ (Value: "what information do I offer?") Where Wᵠ, Wᴷ ∈ ℝ^{dₖ × d} and Wᵛ ∈ ℝ^{dᵥ × d} Q, K, V projections — separate learned matrices for each

Attention Computation — Full Diagram

Scaled Dot-Product Attention: Attention(Q,K,V) = softmax(QKᵀ/√dₖ) · V
# Self-Attention from scratch in PyTorch
import torch
import torch.nn.functional as F
import math

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q, K: (batch, seq, d_k)
    V:    (batch, seq, d_v)
    Returns: (batch, seq, d_v), attn_weights: (batch, seq, seq)
    """
    d_k = Q.size(-1)
    
    # Attention scores: (batch, seq, seq)
    scores = torch.bmm(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
    
    # Apply mask (e.g. causal mask for decoder: -inf for future positions)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))
    
    # Softmax to get attention weights
    attn_weights = F.softmax(scores, dim=-1)  # (batch, seq, seq)
    
    # Weighted sum of values
    output = torch.bmm(attn_weights, V)  # (batch, seq, d_v)
    return output, attn_weights

Autoencoders → VAEs → Diffusion

Regular Autoencoder

Encoder compresses input x → z (bottleneck). Decoder reconstructs x̂ from z. Train to minimize ‖x - x̂‖². Problem: the learned latent space z has no structure — you can't sample new meaningful points from it.

Variational Autoencoders (VAEs)

Instead of encoding to a point z, encode to a Gaussian distribution N(μ(x), σ²(x)). The decoder samples z ~ N(μ,σ²) and reconstructs. This regularizes the latent space to be smooth and structured.

VAE ELBO (Evidence Lower BOund — what we maximize): ELBO = E_{z~q(z|x)}[log p(x|z)] - D_KL(q(z|x) || p(z)) = Reconstruction term - Regularization term Reconstruction: how well does the decoder reconstruct x from sampled z? KL term: how close is q(z|x) = N(μ,σ²) to prior p(z) = N(0,I)? For Gaussian q(z|x) = N(μ,σ²): KL(N(μ,σ²) || N(0,I)) = -½ Σⱼ (1 + log σⱼ² - μⱼ² - σⱼ²) VAE training objective

The Reparameterization Trick

Problem: z ~ N(μ,σ²) is a stochastic operation → can't backpropagate through it Reparameterize: z = μ + σ ⊙ ε where ε ~ N(0,I) Now: - ε is just random noise (not a parameter — no gradient needed) - μ and σ are deterministic functions of x → gradients flow through them - z is still distributed as N(μ,σ²) ✓ Without trick: μ ← (encoder) → (sample) → z [backprop blocked at sample] With trick: μ ← (encoder) → z = μ + σε [backprop flows through μ, σ] The trick that makes VAE training possible 
# VAE in PyTorch (key parts)
class VAE(nn.Module):
    def encode(self, x):
        h = self.encoder(x)
        mu = self.fc_mu(h)       # mean of latent distribution
        log_var = self.fc_var(h)  # log variance (unconstrained)
        return mu, log_var

    def reparameterize(self, mu, log_var):
        std = torch.exp(0.5 * log_var)  # σ = exp(0.5 * log σ²)
        eps = torch.randn_like(std)       # ε ~ N(0,I)
        return mu + eps * std             # z = μ + σ⊙ε

    def vae_loss(recon_x, x, mu, log_var):
        recon_loss = F.binary_cross_entropy(recon_x, x, reduction='sum')
        kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())
        return recon_loss + kl_loss

Diffusion Models — Full Derivation

DDPM: Forward process (add noise) and Reverse process (learned denoising)

DDPM Full Derivation

Forward process — closed form (can jump to any t directly): q(x_t | x_0) = N(x_t; √ᾱ_t · x_0, (1-ᾱ_t)·I) where αₜ = 1-βₜ, ᾱₜ = ∏_{s=1}^{t} αₛ So: x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε, ε ~ N(0,I) Reverse process — what the U-Net learns: p_θ(x_{t-1}|x_t) = N(x_{t-1}; μ_θ(x_t,t), Σ_θ(x_t,t)) DDPM training simplification (Ho et al. 2020): Instead of learning μ_θ directly, predict the noise ε_θ(x_t,t): μ_θ = (1/√αₜ)(x_t - βₜ/√(1-ᾱₜ) · ε_θ(x_t,t)) Loss (simplified): L_simple = E_{t,x_0,ε}[‖ε - ε_θ(√ᾱ_t·x_0 + √(1-ᾱ_t)·ε, t)‖²] = "U-Net: given noisy image at time t, predict the noise that was added" DDPM — elegant reduction to noise prediction

RLHF — Reinforcement Learning from Human Feedback

  1. Supervised Fine-Tuning (SFT): Start from a pretrained LLM. Fine-tune on (prompt, high-quality response) pairs curated by humans. Creates a baseline "helpful" model.
  2. Reward Model (RM): Show annotators multiple completions for the same prompt. Rank them by preference. Train a reward model to predict which response humans prefer. Output: a scalar "reward" score.
  3. PPO (Proximal Policy Optimization): Use the reward model as a reward signal. Optimize the LLM policy using RL to maximize expected reward, with a KL penalty to prevent the model from drifting too far from the SFT baseline (prevents reward hacking).
📌 Why RLHF Changed Everything

A model trained purely on next-token prediction optimizes to predict internet text well — including toxic, false, and unhelpful text. RLHF teaches the model that the goal is "be helpful, harmless, and honest." This is what transformed GPT-3 into ChatGPT. DPO (Direct Preference Optimization, 2023) offers a simpler alternative that skips the RL step and directly optimizes on preference pairs.

PEFT: LoRA & QLoRA

LoRA (Low-Rank Adaptation): Original weight: W₀ ∈ ℝ^{d×d} — frozen during fine-tuning Update: ΔW = B · A, B ∈ ℝ^{d×r}, A ∈ ℝ^{r×d} where r << d Forward: h = W₀x + ΔWx = W₀x + BAx Only A, B are trained. Rank r = 4,8,16 typical (vs full d=4096 etc.) Motivation from SVD: most weight updates during fine-tuning are low-rank i.e., ΔW has most of its "energy" in a few singular vectors Parameter reduction: 2×r×d vs d×d d=4096, r=8: 65,536 vs 16,777,216 — 256× fewer trainable params LoRA — fine-tune large models efficiently

QLoRA: LoRA applied on top of a 4-bit quantized base model. Enables fine-tuning a 70B model on a single consumer GPU (24GB VRAM). Made open-source fine-tuning of large models democratically accessible.

RAG — Retrieval-Augmented Generation

Problem: LLMs have a knowledge cutoff and can hallucinate. Solution: retrieve relevant documents at inference time and include them in the context.

  1. Indexing: Encode a corpus into dense vector embeddings (via an embedding model like BERT or E5). Store in a vector database (Pinecone, Chroma, Weaviate, FAISS).
  2. Retrieval: Encode the user query. Find top-k most similar document embeddings (cosine similarity / approximate nearest neighbor search).
  3. Generation: Prepend the retrieved documents to the user query in the LLM prompt. Generate a response grounded in the retrieved content.

RAG is central to almost all production LLM applications (enterprise search, documentation Q&A, customer support). As an FDE, you will deploy RAG systems constantly.

Fine-tuningContinue training a pretrained model on task-specific data. Much cheaper than pretraining.
Zero-shotModel performs a task with no examples — just a description. Works for large models.
Few-shotGive k examples in the prompt. In-context learning — no weight updates.
CoT (Chain of Thought)Prompt model to show reasoning steps before final answer. Improves multi-step tasks significantly.
KV CacheCache computed Key/Value vectors during generation — avoid recomputing for already-generated tokens.
QuantizationReduce weight precision (FP32→INT8→INT4). Massive memory savings, minor accuracy loss.
Perplexityppl = 2^H where H = cross-entropy. Lower = model better models the data. Typical LLM: 3-10 on test text.
SoftmaxConverts logit vector to probability distribution summing to 1. Temperature controls sharpness.
Ground truthThe true/correct label for a training or test example. What the model should predict.
InferenceUsing a trained model to make predictions on new data. As opposed to training.
Vector databaseDatabase for storing and querying dense vector embeddings via approximate nearest neighbor (ANN) search. Used in RAG.
LoRA rank rThe dimensionality of the low-rank matrices in LoRA. r=4,8,16 are typical. Higher r = more capacity but more parameters.
Latent spaceThe compressed representation space inside a model (e.g., VAE bottleneck, between encoder and decoder).
Mixture of Experts (MoE)Architecture with many "expert" FFN layers; routing mechanism selects k experts per token. Scale model size without proportionally scaling compute.
