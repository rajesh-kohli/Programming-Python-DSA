Chapter 00 · Start Here

The Big Picture

The #1 reason people get lost in ML/DL is they encounter terms — "transformer," "gradient," "attention," "latent space" — without knowing which layer of the field those terms live in. Let's build the map first.

How the Fields Nest

AI → ML → DL → Generative AI — Field Nesting
🤖 Artificial Intelligence

Any technique making computers behave intelligently. Includes rule-based systems, search, planning, expert systems, and ML. Exists since 1950s (Turing, McCarthy).

📊 Machine Learning

Systems that learn from data instead of being explicitly programmed. You provide labeled examples; the algorithm finds the pattern. Flourished in 1990s–2000s.

🧠 Deep Learning

ML using multi-layer neural networks. Key insight: hierarchical feature learning — early layers learn edges, later layers learn faces. Dominant since AlexNet (2012).

✨ Generative AI

DL models that can create new content — text, images, code, audio. Subset of DL. Includes LLMs (GPT, Claude), diffusion models (Stable Diffusion), VAEs. Public breakthrough: ChatGPT (2022).

Adjacent fields that intersect constantly: NLP (language understanding/generation — now almost entirely DL), Computer Vision (image/video — CNNs then ViTs), Reinforcement Learning (agents learning from reward signals — used in game AI and RLHF for LLMs), Data Science (your background — data pipelines, analysis, visualization, and ML).

Historical Evolution — Why Each Step Happened

Every major architecture was invented to fix a specific failure in the previous one. The why is as important as the what.

1943
McCulloch-Pitts Neuron — First mathematical neuron. Binary threshold logic. No learning. "A neuron fires if inputs exceed threshold." Problem: cannot learn.
1958
Perceptron (Rosenblatt) — First trainable single-layer neuron. Learned linear boundaries. Huge hype. Fatal flaw: cannot solve XOR (non-linear). Led to 15-year "AI Winter."
1986
Backpropagation (Rumelhart, Hinton) — Made multi-layer nets trainable. Stack layers → solve XOR → solve complex tasks. The chain rule applied backward through a network. Revived neural nets.
1989–98
LeNet (LeCun) — First CNN — Handwritten digit recognition (MNIST). Introduced convolution + pooling. Key insight: spatial structure matters. Too slow on available hardware for practical use.
1997
LSTM (Hochreiter & Schmidhuber) — Plain RNNs suffered from vanishing gradients (couldn't learn long-range dependencies). LSTM's gating mechanism let gradients flow unchanged. Enabled language modeling, speech, time series.
2012
AlexNet — The Revolution — Deep CNN on GPU cuts ImageNet error 26%→16% (vs. next best at 26%). GPUs + ReLU + Dropout + Big data = DL takes over. Every major company scrambles for deep learning talent.
2014
GANs (Goodfellow) + Seq2Seq — GANs: adversarial training for generation. Seq2Seq: encoder-decoder architecture for translation using RNNs — but bottlenecks on long sentences.
2015
Attention Mechanism + ResNet — Bahdanau attention lets decoder look at any encoder state (fixes bottleneck). ResNet's skip connections enable 150+ layer networks without degradation.
2017
"Attention Is All You Need" — The Transformer — Eliminate RNNs entirely. Pure attention-based architecture. Fully parallelizable — train on 8 GPUs in days, not months. Scaled to any data size. Foundation of everything modern.
2018
BERT + GPT-1 — Transfer Learning Era — Pretrain on massive unlabeled text → fine-tune on task. BERT: bidirectional encoding for understanding. GPT: autoregressive generation. Changed NLP permanently.
2020
GPT-3 — Scale Changes Everything — 175B parameters. In-context few-shot learning: describe the task in the prompt, no fine-tuning needed. Emergent capabilities. Foundation model era begins.
2020–22
Diffusion Models — DDPM, DALL-E, Stable Diffusion — New generative paradigm: learn to reverse a gradual noising process. Outperforms GANs on image quality + diversity. Open-sourced by Stability AI.
2022–23
ChatGPT / Claude / GPT-4 — RLHF Era — Reinforcement Learning from Human Feedback aligns LLMs to be helpful, harmless, honest. 100M users in 2 months. AI enters mainstream.
2024–26
Reasoning Models, Agents, Multimodal — o1/o3, DeepSeek-R1: "think before answering" (chain-of-thought at inference). Claude/GPT-4o: vision+text+audio natively. Mamba, MoE: efficient architectures challenging dense transformers. Agentic loops becoming real.
Chapter 01 · Mathematical Foundations

Mathematical Foundations

You need fluency in three areas: linear algebra (the language), calculus (the engine), and probability (the framework). Not PhD-level — but you must be able to read a paper's math and know what it's saying.

Vectors & Matrices

Vectors

A vector is an ordered list of numbers. In ML: a data point, a word embedding, the output of a layer. A 4D vector: x = [0.2, 0.8, -0.5, 1.3]. Think of it geometrically as a point in multi-dimensional space, or an arrow with direction and magnitude.

Dot product: a · b = Σᵢ aᵢbᵢ. Measures "alignment" between two vectors. If a·b is large, the vectors point in similar directions. This is the core operation in attention mechanisms.

a = [1, 2, 3], b = [4, 0, -1] a · b = (1×4) + (2×0) + (3×-1) = 4 + 0 - 3 = 1 Dot product — core of attention and similarity computation

Matrices

A matrix is a 2D array — rows and columns. Notation: W ∈ ℝⁿˣᵐ means n rows, m columns.

Matrix Multiplication — The Core Operation

If W is (n×m) and x is (m×1), then y = Wx is (n×1). Each output element yᵢ = row i of W dotted with x. This is how every neural network layer works: linear transformation of data.

W = [[2, 1], [0, 3], [-1, 2]] (3×2 matrix) x = [4, -1] (2-vector) y = W·x: y₁ = 2×4 + 1×(-1) = 7 y₂ = 0×4 + 3×(-1) = -3 y₃ = (-1)×4 + 2×(-1) = -6 y = [7, -3, -6] (3-vector) Matrix-vector multiplication: transforms a 2D input to 3D output space
💡 Why this matters everywhere

Every "layer" in a neural network is: y = W·x + b. W is the weight matrix (learned), x is the input, b is a bias vector (learned), y is the output. A 512→256 hidden layer has W of shape (256×512) — that's 131,072 learned weights in one layer. Training = finding the right values for all W and b matrices.

Calculus & Gradients

What is a Gradient?

The gradient of a scalar function f(w₁, w₂, ..., wₙ) is the vector of partial derivatives: ∇f = [∂f/∂w₁, ∂f/∂w₂, ..., ∂f/∂wₙ]. It points in the direction of steepest increase. To minimize f (reduce the loss), we move in the negative gradient direction.

Chain Rule — The Engine of Backprop

For composed functions: if z = f(y) and y = g(x), then dz/dx = (dz/dy) · (dy/dx). A neural network is a composition of many functions. Chain rule lets you compute how the loss L changes with respect to any weight, no matter how deep it is.

📐 Derivation: Chain Rule Through a Simple Network

Forward pass: x → z₁ = W₁x → a₁ = ReLU(z₁) → z₂ = W₂a₁ → L = MSE(z₂, y)

Step 1: Loss gradient w.r.t. z₂∂L/∂z₂ = 2(z₂ - y)/n (derivative of MSE)
Step 2: Gradient w.r.t. W₂∂L/∂W₂ = (∂L/∂z₂) · aᵀ₁ (a₁ is the activation going IN to W₂)
Step 3: Gradient flowing back through W₂∂L/∂a₁ = Wᵀ₂ · (∂L/∂z₂) (transpose of W₂ times upstream gradient)
Step 4: Gradient through ReLU∂L/∂z₁ = (∂L/∂a₁) ⊙ 𝟙[z₁ > 0] (1 where ReLU was active, 0 where it was off)
Step 5: Gradient w.r.t. W₁∂L/∂W₁ = (∂L/∂z₁) · xᵀ (same pattern — upstream gradient × local input)

Notice the pattern: to get gradient of W, multiply (upstream gradient) × (input to that layer). To pass gradient to previous layer, multiply (upstream gradient) × (transposed weight matrix). This is exactly what PyTorch's autograd does — automatically, for any arbitrary network.

Gradient Descent — The Update Rule

Gradient Descent on a Loss Surface
Gradient Descent: w ← w − η · ∂L/∂w Mini-batch SGD: w ← w − η · (1/|B|) · Σᵢ∈B ∂Lᵢ/∂w η = learning rate (step size). Typical values: 0.001 – 0.1

Maximum Likelihood Estimation (MLE)

The mathematical foundation of training. Given data D = {x₁...xₙ}, find parameters θ that maximize the probability of observing the data: θ* = argmax_θ P(D|θ). Taking log (monotonic, so same argmax): θ* = argmax_θ Σᵢ log P(xᵢ|θ). Minimizing cross-entropy loss is exactly MLE for a classification model — they are equivalent.

Classical Machine Learning

Types of Learning

📚 Supervised

Labeled input→output pairs. Model learns mapping by minimizing error. Examples: spam detection, house price prediction, image classification, fraud detection.

🔍 Unsupervised

No labels. Find hidden structure. Examples: customer segmentation (K-Means), dimensionality reduction (PCA), anomaly detection, topic modeling.

🎮 Reinforcement

Agent takes actions, receives rewards/penalties, learns to maximize total reward. Examples: AlphaGo, robotics, game playing, RLHF for LLM alignment.

🔄 Self-Supervised

Supervision from data itself. GPT: predict next word. BERT: predict masked word. No human labeling. Why this changed everything: enables pretraining on internet-scale text.

Bias-Variance Tradeoff — The Fundamental Tension

Bias: Error from overly simple models that miss patterns. High bias = underfitting (linear model on non-linear data).
Variance: Error from models too sensitive to training noise. High variance = overfitting (memorizes training set, fails on new data).
Total Error = Bias² + Variance + Irreducible Noise.

📌 The Practical Rule

Classical ML: increase model complexity → lower bias, higher variance. Deep learning at scale partially breaks this: very large neural networks can have high capacity yet still generalize well when trained with regularization, dropout, data augmentation, and massive data. This is the "double descent" phenomenon — test error decreases, increases, then decreases again as model size grows.

Chapter 03 · Neural Networks

Neural Networks — From Perceptron to Deep Networks

The Biological Analogy

A biological neuron: receives signals from dendrites → sums weighted inputs → fires if sum exceeds threshold → sends signal via axon. Artificial neuron: output = activation(Σᵢ wᵢxᵢ + b). Same idea: weighted sum → non-linear threshold.

Multi-Layer Perceptron (MLP) — Forward Pass

Multi-Layer Perceptron (3 → 4 → 4 → 2)

Forward pass equations for each layer:

Layer 1: z₁ = W₁·x + b₁ (linear transform, W₁ shape: 4×3) a₁ = ReLU(z₁) (activation, shape: 4) Layer 2: z₂ = W₂·a₁ + b₂ (W₂ shape: 4×4) a₂ = ReLU(z₂) Output: z₃ = W₃·a₂ + b₃ (W₃ shape: 2×4) ŷ = softmax(z₃) (probability over 2 classes) Every layer: linear transformation (W·x+b) → non-linearity (activation)

Why are multiple layers necessary? A single layer can only draw linear decision boundaries. Two layers can approximate any continuous function (Universal Approximation Theorem — Cybenko, 1989). Deeper networks learn hierarchical representations more efficiently — rather than memorizing the full function, they factorize it into reusable components.

Backpropagation — Full Derivation

The algorithm that makes training neural networks possible. Invented by Rumelhart, Hinton & Williams (1986) — though the mathematical roots go back further. It's simply the chain rule of calculus applied systematically through a computation graph.

Worked Derivation: 2-layer Network

Setup

Input x ∈ ℝⁿ, Target y ∈ ℝ¹ (regression). Network: x → z₁=W₁x+b₁ → a₁=σ(z₁) → ẑ=W₂a₁+b₂ → L = ½(ẑ-y)². We want: ∂L/∂W₂, ∂L/∂b₂, ∂L/∂W₁, ∂L/∂b₁.

∂L/∂ẑ — derivative of loss w.r.t. output∂L/∂ẑ = (ẑ - y) ← derivative of ½(ẑ-y)²
∂L/∂W₂ — gradient of output layer weights∂L/∂W₂ = (∂L/∂ẑ) · a₁ᵀ = (ẑ-y) · a₁ᵀ ← outer product (shape: 1×hidden)
∂L/∂b₂ — gradient of output layer bias∂L/∂b₂ = (∂L/∂ẑ) = (ẑ-y) ← same, bias doesn't multiply
∂L/∂a₁ — gradient flowing back through W₂∂L/∂a₁ = W₂ᵀ · (∂L/∂ẑ) = W₂ᵀ · (ẑ-y) ← transpose times upstream gradient
∂L/∂z₁ — gradient through sigmoid σ∂σ/∂z = σ(z)(1-σ(z)) → ∂L/∂z₁ = (∂L/∂a₁) ⊙ σ(z₁)(1-σ(z₁)) ← element-wise multiply
∂L/∂W₁ — gradient of hidden layer weights∂L/∂W₁ = (∂L/∂z₁) · xᵀ ← same pattern: upstream gradient × input
💡 The Pattern

Every layer follows the same pattern: (1) receive gradient from above, (2) compute gradient for this layer's weights = upstream_gradient × this_layer's_input, (3) pass gradient further down = transpose(this_layer's_weights) × upstream_gradient. PyTorch's autograd does all of this automatically by building a computation graph during the forward pass.

SGD and Momentum

SGD: w ← w - η·∇L Momentum: v ← βv + ∇L w ← w - η·v (β≈0.9: 90% of old velocity retained) Momentum: accumulates velocity in consistent directions, dampens oscillations

Adam (Adaptive Moment Estimation, 2015)

Maintains per-parameter adaptive learning rates. First moment (mean of gradients, like momentum) and second moment (variance of gradients). Parameters getting consistently large gradients get smaller effective learning rates.

m_t = β₁·m_{t-1} + (1-β₁)·g_t (first moment, β₁=0.9) v_t = β₂·v_{t-1} + (1-β₂)·g_t² (second moment, β₂=0.999) m̂_t = m_t/(1-β₁ᵗ) (bias correction — important early in training) v̂_t = v_t/(1-β₂ᵗ) (bias correction) w_t = w_{t-1} - η · m̂_t/(√v̂_t + ε) (ε=1e-8 for numerical stability) Adam: combines momentum (m) with per-parameter adaptive rates (v). Near-universal choice.

AdamW — Why it Matters

In Adam with L2 regularization, weight decay is added to the gradient: g = ∇L + λw. The problem: this decay then gets scaled by the adaptive learning rate 1/√v̂ — meaning frequently updated parameters get less weight decay than infrequently updated ones. This is incorrect.

AdamW decouples weight decay from the gradient update:

Adam+L2: w ← w - η · (m̂/(√v̂+ε) + λw) ← decay scaled by adaptive rate ❌ AdamW: w ← (1 - η·λ)·w - η · m̂/(√v̂+ε) ← decay applied directly ✓ AdamW is the standard for Transformers. Better generalization than Adam+L2.

Regularization

L1 and L2 Regularization

Add a penalty to the loss function:

L1: L_total = L_data + λ·Σ|wᵢ| ← encourages sparse weights (many exact zeros) L2: L_total = L_data + λ·Σwᵢ² ← shrinks weights toward zero, rarely exact zero λ = regularization strength (hyperparameter). Weight decay in AdamW implements L2.

Dropout

During training, randomly zero out neurons with probability p (typically 0.1–0.5). Forces the network to develop redundant representations — no single neuron can be relied upon. At inference: scale outputs by (1-p) to preserve expected value (or use "inverted dropout" — scale during training).

# PyTorch
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(512, 256), nn.ReLU(), nn.Dropout(p=0.3),
            nn.Linear(256, 128), nn.ReLU(), nn.Dropout(p=0.3),
            nn.Linear(128, 10)
        )
    def forward(self, x):
        return self.layers(x)  # dropout auto-disabled in model.eval() mode

Weight Initialization & Learning Rate Schedules

Weight Initialization

Why it matters: If all weights are zero, all neurons compute the same thing (symmetry problem). If weights are too large, activations saturate; too small, signals vanish. Good initialization keeps variance of activations roughly constant across layers.

Learning Rate Schedules

A constant learning rate is rarely optimal. Most modern training uses a warmup + decay schedule:

  • Linear warmup: Start from near-zero, ramp up to target LR over first 1–10% of training. Prevents large early updates before model is stable.
  • Cosine decay: After warmup, follow a cosine curve from peak LR to near-zero. Most common for LLMs.
  • Step decay: Reduce LR by factor (e.g., ×0.1) at fixed epochs. Used in ResNet training.
  • OneCycleLR: Fast.ai's policy — warmup then decay in one cycle. Very fast convergence.
Chapter 05 · Convolutional Neural Networks

Convolutional Neural Networks (CNNs)

The dominant architecture for image/video from 2012–2020. Still foundational. CNNs encode two powerful inductive biases about images: locality (nearby pixels are related) and translation equivariance (a cat looks like a cat whether it's top-left or bottom-right). These reduce parameters dramatically vs a dense MLP and often generalize better.

The Core Insight: Weight Sharing

A 224×224 RGB image has 224×224×3 = 150,528 pixels. If you used a dense layer with 1,000 hidden units, you'd need 150 million parameters — just for the first layer! CNNs instead learn a small filter (e.g., 3×3×3 = 27 parameters) and apply it everywhere. A filter for detecting vertical edges looks the same in the top-left as the bottom-right — you don't need separate edge-detectors for every location.

Convolution — Worked Example

3×3 Filter Sliding Across a 5×5 Input → 3×3 Feature Map

Each filter learns to detect a specific pattern. With 32 filters in the first layer, you get 32 different feature maps — each highlighting something different (edges in different directions, color gradients, etc.). Deeper layers combine these to detect complex shapes and eventually objects.

Receptive Fields

The receptive field of a neuron = the region of the original input that can influence its activation. Crucially: it grows with depth. With 3×3 filters and stride=1:

After layer 1 (3×3 filter): receptive field = 3×3 After layer 2 (3×3 filter): receptive field = 5×5 After layer k (3×3 filters): receptive field = (2k+1)×(2k+1) With stride=2 at each layer: receptive field grows much faster Deeper networks "see" larger context — enabling detection of global features

Effective receptive field: In practice, not all pixels in the theoretical receptive field contribute equally. Central pixels have more influence paths; edge pixels fewer. The effective RF is often much smaller than the theoretical one — an important practical consideration.

Dilated (Atrous) Convolutions

Insert gaps (rate r) between filter elements. A 3×3 filter with dilation rate=2 covers a 5×5 area but only has 9 actual weights. Exponentially grows receptive field without increasing parameters or losing resolution. Used in: DeepLab (semantic segmentation), WaveNet (audio generation), TemporalConvNet (time series).

ResNet & Skip Connections — Why They Work

Without skip connections, deep networks (20+ layers) actually perform worse than shallow ones — not from overfitting, but from optimization failure. The gradient degrades as it propagates back through many layers.

Standard layer: y = F(x, W) ← learn full mapping Residual block: y = F(x, W) + x ← learn residual F(x,W) = desired - identity ∂L/∂x = ∂L/∂y · (∂F/∂x + 1) ← gradient always has +1 term! Skip connection guarantees gradient can flow unchanged (∂L/∂x ≥ ∂L/∂y). No vanishing.

If a layer is unhelpful, it can easily learn F(x,W)=0 (making the block an identity). This is much easier than learning the identity from scratch. Skip connections are now universal — every Transformer uses them as "Add & Norm" layers.

Recurrent Neural Networks & Sequence Models

Images have spatial structure — nearby pixels relate. Text, audio, time series have temporal structure — the order of elements matters fundamentally. "The dog bit the man" ≠ "The man bit the dog." Standard MLPs and CNNs can't handle variable-length sequences with order-dependent meaning. Enter RNNs.

The RNN — Unrolled Through Time

RNN Unrolled: Processing "cat sat on mat"
hₜ = tanh(Wₓₓ · xₜ + Wₕₕ · hₜ₋₁ + b) yₜ = Wₕᵧ · hₜ (if producing output at each step) Same W matrices reused at every step — this is temporal weight sharing

Vanishing Gradient — Root Cause

During backpropagation through time (BPTT), gradients must travel backward through T time steps. At each step, you multiply by Wₕₕ and the derivative of tanh (which is ≤ 1). After T steps:

∂L/∂h₁ = ∏ₜ₌₂ᵀ (∂hₜ/∂hₜ₋₁) · (∂L/∂hₕ) = ∏ₜ₌₂ᵀ diag(tanh'(hₜ₋₁)) · Wₕₕ If largest eigenvalue of Wₕₕ < 1: gradient shrinks exponentially → vanishes If largest eigenvalue of Wₕₕ > 1: gradient grows exponentially → explodes This is why plain RNNs can only reliably use ~10-20 steps of history

LSTM — Long Short-Term Memory (Hochreiter & Schmidhuber, 1997)

LSTM Cell — Gates and Cell State Highway

LSTM Gate Equations — Fully Derived

Input concatenation[hₜ₋₁, xₜ] — concatenate previous hidden state with current input
Forget Gate — what to erase from cell statefₜ = σ(Wf·[hₜ₋₁, xₜ] + bf) fₜ ∈ (0,1): 0=forget completely, 1=keep completely
Input Gate — what new info to writeiₜ = σ(Wᵢ·[hₜ₋₁, xₜ] + bᵢ) iₜ ∈ (0,1): how much of candidate to write
Candidate cell state — what to potentially writeC̃ₜ = tanh(Wc·[hₜ₋₁, xₜ] + bc) C̃ₜ ∈ (-1,1): candidate values for cell state
Cell state update — additive, not multiplicative!Cₜ = fₜ ⊙ Cₜ₋₁ + iₜ ⊙ C̃ₜ Erase old, add new. Additive update = gradient highway
Output Gate — what to expose as hidden stateoₜ = σ(Wo·[hₜ₋₁, xₜ] + bo) oₜ ∈ (0,1)
Hidden state outputhₜ = oₜ ⊙ tanh(Cₜ) Filter cell state through output gate
🔑 Why LSTMs Solve Vanishing Gradients

The key: Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ is an additive update. When you backpropagate through time: ∂Cₜ/∂Cₜ₋₁ = fₜ (the forget gate value). Unlike the RNN's multiplicative Wₕₕ, this can be close to 1.0 (meaning "don't forget this information"). The gradient can flow through the cell state for hundreds of steps without vanishing. This is the "constant error carousel" — Hochreiter's original description.

GRU (Gated Recurrent Unit) — Cho et al. 2014

Simplified LSTM: merges forget and input gates into one update gate, eliminates separate cell state. Fewer parameters, often comparable performance, faster training.

zₜ = σ(Wz·[hₜ₋₁, xₜ]) (update gate: how much to update hidden state) rₜ = σ(Wr·[hₜ₋₁, xₜ]) (reset gate: how much of past to forget) h̃ₜ = tanh(W·[rₜ⊙hₜ₋₁, xₜ]) (candidate hidden state) hₜ = (1-zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ (interpolate old and new — like LSTM cell update) GRU: 2 gates vs LSTM's 3. Same intuition, fewer parameters. Good for smaller datasets.

Chapter 07 · Attention Mechanisms

Attention Mechanisms

The Seq2Seq Bottleneck Problem

Seq2Seq (Sutskever et al., 2014): encode an entire input sequence into a single fixed-size vector (the "context vector"), then decode. For "The agreement on the European Economic Area was signed in August 1992" → French, the entire sentence must fit in a ~256-dimensional vector. Long sentences: crucial early information gets "forgotten."

Bahdanau Attention (2015) — The Solution

At each decoding step, instead of using only the final encoder state, compute a weighted combination of all encoder states. The weights are learned based on compatibility between the current decoder state and each encoder state.

Score: eₜᵢ = vᵀ tanh(Wₛ·sₜ₋₁ + Wₕ·hᵢ) (alignment score: decoder state vs encoder state i) Weights: αₜᵢ = softmax(eₜᵢ) = exp(eₜᵢ)/Σⱼexp(eₜⱼ) (attention distribution, sums to 1) Context: cₜ = Σᵢ αₜᵢ · hᵢ (weighted sum of encoder states) Decode: sₜ = f(sₜ₋₁, yₜ₋₁, cₜ) (use context for this step's generation) Bahdanau: additive/concatenation attention. Learned alignment — the model discovers which input words to focus on when generating each output word.

A key result: the learned attention weights form interpretable alignment matrices — when translating "European Economic Area," the French output "zone économique européenne" attends to those specific input words in the right order. Attention provides interpretability as a byproduct.

Self-Attention — The Transformer's Core

Regular attention: decoder attends to encoder. Self-attention: each position in a sequence attends to all other positions in the same sequence. This lets representations be context-aware — "bank" in "river bank" vs "bank account" gets different representations based on surrounding words.

Three projections of the input X:

Q = X·Wᵠ (Query: "what am I looking for?") K = X·Wᴷ (Key: "what do I contain?") V = X·Wᵛ (Value: "what do I offer?") Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V Why √dₖ scaling? For high-dimensional vectors (dₖ=64), dot products grow in magnitude (~√dₖ), pushing softmax into saturation (near-zero gradients). Dividing by √dₖ stabilizes training. Scaled Dot-Product Attention — foundation of all modern Transformers

Worked Numerical Example

Self-Attention: 3 tokens, dₖ = 2 (simplified)

Sequence: ["I", "love", "cats"] with embeddings (2D for simplicity)

Input embeddings (X) X = [[1, 0], ("I") [0, 1], ("love") [1, 1]] ("cats") Weight matrices (learned): Wᵠ = Wᴷ = Wᵛ = I (identity, for simplicity) So Q = K = V = X (same in this simplified example)
Compute scores: QKᵀ QKᵀ = X·Xᵀ: "I"·"I"=1, "I"·"love"=0, "I"·"cats"=1 "love"·"I"=0, "love"·"love"=1, "love"·"cats"=1 "cats"·"I"=1, "cats"·"love"=1, "cats"·"cats"=2 Score matrix = [[1,0,1],[0,1,1],[1,1,2]] / √2 ≈ [[0.7,0,0.7],[0,0.7,0.7],[0.7,0.7,1.4]]
Apply softmax to each row → attention weights Row 1 (for "I"): softmax([0.7, 0, 0.7]) ≈ [0.42, 0.15, 0.42] ← "I" attends mostly to "I" and "cats" Row 2 (for "love"): softmax([0, 0.7, 0.7]) ≈ [0.15, 0.42, 0.42] ← "love" attends mostly to "love","cats" Row 3 (for "cats"): softmax([0.7,0.7,1.4]) ≈ [0.25, 0.25, 0.50] ← "cats" attends most to itself
Output: attention weights × V Output_"I" = 0.42×[1,0] + 0.15×[0,1] + 0.42×[1,1] = [0.84, 0.57] Now "I" has information from "cats" in its representation — context-aware!
💡 The Database Retrieval Analogy

Think of attention as a soft, differentiable database lookup: Query = your search term. Keys = "tags" on stored documents. Values = actual content of documents. Dot(Q,K) measures relevance. Softmax normalizes to weights. ΣαᵢVᵢ retrieves a weighted blend. Unlike hard lookup returning one document, attention returns a smooth mixture — and the whole thing is differentiable so you can learn the best Q, K, V projections through gradient descent.

Chapter 08 · The Transformer

"Attention Is All You Need" — The Transformer (2017)

Vaswani et al. (Google Brain). 8 authors. English-German/French translation benchmarks. The claim: you can eliminate RNNs entirely and build sequence models using only attention. This enables full parallelization (RNNs process sequentially) and scales to massive data and compute. The most cited AI paper of the decade.

Transformer Architecture — Encoder (left) and Decoder (right)

Encoder Stack — Deep Dive

6 identical encoder layers (in the original paper). Each layer has two sub-layers:

  1. Multi-head self-attention: every position attends to every other position in the same sequence. Produces contextually-enriched representations.
  2. Position-wise Feed-Forward Network (FFN): Two linear layers with GELU/ReLU: FFN(x) = Linear₂(GELU(Linear₁(x))). Applied identically to each position independently. The FFN layers are where most of the model's "knowledge" is thought to be stored (they're much larger — 4× the hidden dimension typically).

Each sub-layer is wrapped with residual (skip) connection + Layer Norm: output = LayerNorm(x + sublayer(x)).

Decoder Stack — Deep Dive

6 identical decoder layers. Three sub-layers:

  1. Masked self-attention: positions can only attend to previous (left) positions. The "mask" zeros out all future attention scores before softmax. This enforces causality — when generating word 5, you can't look at words 6, 7, 8... (you haven't generated them yet).
  2. Cross-attention (encoder-decoder attention): Queries come from the decoder's previous sub-layer. Keys and Values come from the encoder's final output. This is where the Bahdanau attention idea is incorporated — the decoder reads the encoded source sequence.
  3. FFN + Add & Norm: same as encoder.

Multi-Head Attention — Parallel Attention Subspaces

Instead of one attention function with full dimensionality, run h=8 parallel attention "heads" with lower dimension (dₖ = d_model/h = 512/8 = 64 each). Concatenate results, project back.

For head i: headᵢ = Attention(Q·Wᵢᵠ, K·Wᵢᴷ, V·Wᵢᵛ) (Wᵢᵠ, Wᵢᴷ, Wᵢᵛ ∈ ℝ^{512×64}) MultiHead(Q,K,V) = Concat(head₁,...,head₈) · Wᴼ (Wᴼ ∈ ℝ^{512×512}) Each head learns different relationship types. One might learn syntax, another semantics, another coreference.

Why multiple heads work: In a single attention head, after the softmax, the output is a weighted average of values — it can only "look at one thing" at a time (the weighted average loses the multi-modal structure). Multiple heads can attend to different things simultaneously and capture richer interactions.

Positional Encoding — Adding Order Information

Self-attention is permutation-invariant by design — if you shuffle input tokens, the output is the same (just shuffled). But language is order-dependent. Positional encoding injects position information.

Original sinusoidal encoding: PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model)) Intuition: Different frequencies — low-i dimensions encode large-scale position information, high-i dimensions encode fine-grained position. Like binary representation of numbers. The model can attend to relative positions by noting: PE(pos+k) = function(PE(pos)). Modern models use RoPE (Rotary Position Embeddings) — encode position as rotation in embedding space. Better for long contexts.

Chapter 10 · Generative Models

Generative Models: VAEs, GANs, Diffusion

Generative models learn the distribution of training data well enough to sample new, realistic examples. Three major families, each with different mathematics and tradeoffs.

Autoencoders — The Foundation

Encoder compresses x → z (latent code, bottleneck). Decoder reconstructs z → x̂. Loss = reconstruction error: L = ‖x - x̂‖². Used for compression, denoising, anomaly detection. Problem: the latent space is unstructured — you can't sample from arbitrary z and get realistic outputs.

Variational Autoencoders (VAEs) — Kingma & Welling, 2013

Key change: instead of encoding to a single point z, encode to a probability distribution q(z|x) = N(μ, σ²). Sample z from this distribution. Decode z → x̂. This forces the latent space to be smooth and continuous — you can sample any z near the training distribution and get a realistic output.

VAE Architecture: Encoder → Latent Distribution → Sample → Decoder
ELBO (Evidence Lower BOund) = Evidence we want to maximize: log p(x) ≥ E_q[log p(x|z)] - KL(q(z|x) || p(z)) ↑ reconstruction ↑ regularization term (want high) term (want low — keep q close to N(0,I)) Maximize ELBO = good reconstruction + smooth/structured latent space

The Reparameterization Trick

Problem: we need to backprop through a sampling operation z ~ N(μ, σ²). But sampling is stochastic — no gradient flows through it.

Original (NOT differentiable): z ~ N(μ, σ²) ← can't backprop Reparameterized: z = μ + σ ⊙ ε, ε ~ N(0,I) ← CAN backprop through μ and σ! Gradient: ∂z/∂μ = 1, ∂z/∂σ = ε The randomness (ε) is treated as a fixed external input, not a parameter. This allows gradient-based optimization of μ and σ through the sampling step. 
class VAE(nn.Module):
    def encode(self, x):
        h = self.encoder(x)
        mu = self.fc_mu(h)         # mean of latent distribution
        log_var = self.fc_var(h)   # log variance (for stability)
        return mu, log_var

    def reparameterize(self, mu, log_var):
        std = torch.exp(0.5 * log_var)   # σ = exp(½ log σ²)
        eps = torch.randn_like(std)          # ε ~ N(0,1)
        return mu + eps * std              # z = μ + σε — differentiable!

    def vae_loss(self, x, x_hat, mu, log_var):
        recon = F.mse_loss(x_hat, x, reduction='sum')
        kld = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())
        return recon + kld

Generative Adversarial Networks (Goodfellow et al., 2014)

Two networks competing in a game:

  • Generator G(z): Takes random noise z ~ N(0,I) → produces fake data. Wants to fool D.
  • Discriminator D(x): Takes data → outputs P(real). Wants to distinguish real from fake.
Minimax objective: min_G max_D V(D,G) = E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 - D(G(z)))] ↑ D correctly identifies real ↑ D correctly identifies fake (G wants this term large!) D update: maximize V by improving real/fake discrimination G update: minimize V = maximize log D(G(z)) (trick: instead of minimizing log(1-D(G(z)))) At Nash equilibrium: G produces perfect samples, D outputs 0.5 everywhere (50/50 guess)

Diffusion Models (DDPM) — Ho et al., 2020

The current state-of-the-art for image generation. Elegant idea: train a neural network to reverse a gradual noising process.

Diffusion Forward (Add Noise) and Reverse (Denoise) Process

DDPM — Mathematical Derivation

Forward Process q(xₜ|x₀) — closed form for any t q(xₜ|xₜ₋₁) = N(xₜ; √(1-βₜ)xₜ₋₁, βₜI) (add small noise at each step) Define: ᾱₜ = ∏ₛ₌₁ᵗ (1-βₛ) (cumulative noise schedule) Then: xₜ = √ᾱₜ · x₀ + √(1-ᾱₜ) · ε, ε~N(0,I) (can jump to any step t directly!)
Reverse Process — what we train We train a U-Net ε_θ(xₜ, t) to predict the noise ε that was added to x₀ to get xₜ. Training objective (simplified): L = E_{t,x₀,ε} [‖ε - ε_θ(√ᾱₜx₀ + √(1-ᾱₜ)ε, t)‖²] Just MSE between true noise ε and predicted noise ε_θ — very simple!
Inference (DDPM sampling) Start: xT ~ N(0,I) (pure noise) For t = T, T-1, ..., 1: predicted_noise = ε_θ(xₜ, t) x₀_pred = (xₜ - √(1-ᾱₜ)·predicted_noise) / √ᾱₜ xₜ₋₁ = √ᾱₜ₋₁ · x₀_pred + √(1-ᾱₜ₋₁) · predicted_noise + σₜ · z (z~N(0,I)) Result: a new image generated from random noise — T=1000 denoising steps
🔗 Latent Diffusion (Stable Diffusion)

DDPM in pixel space is expensive (1000 steps × full image resolution). Stable Diffusion's insight: use a VAE to compress images to a latent space (e.g., 512×512×3 → 64×64×4), run the diffusion process in latent space, then decode with the VAE decoder. This is 8× cheaper while producing comparable quality. The VAE encoder/decoder are trained separately. This architecture (Latent Diffusion Models, Rombach et al., 2022) is what powers Stable Diffusion, DALL-E 3, Midjourney, and Sora.

Chapter 11 · Modern Landscape (2026)

The Modern Landscape

Scaling Laws — Why Size Matters

Kaplan et al. (2020, OpenAI): loss follows a power law with N (parameters), D (data), and C (compute). Specifically: L(N,D) ≈ (Nc/N)^αN + (Dc/D)^αD + L∞. This means you can predict performance at scale before training. The Chinchilla paper (Hoffmann et al., 2022) found the optimal compute allocation: for a given compute budget, train a model with N parameters on 20N tokens (not the previous rule of "train on everything available").

📌 Scaling Insight

Llama 3 (Meta, 2024) trains a 8B model on 15 trillion tokens — far more than Chinchilla-optimal for training efficiency, but optimal for inference (a smaller, very well-trained model is cheaper to run at scale). This "inference-optimal" training is increasingly important as serving costs dominate.

RLHF — Reinforcement Learning from Human Feedback

The technique that turned GPT-3 into ChatGPT. A raw language model trained on next-token prediction can generate plausible but unhelpful, harmful, or dishonest text. RLHF aligns it to human values.

  1. SFT (Supervised Fine-Tuning): Collect (prompt, ideal response) pairs from human labelers. Fine-tune the base LLM on these pairs. Gets the model to follow instructions.
  2. Reward Model Training: Show labelers K responses to the same prompt, have them rank them. Train a reward model Rθ(prompt, response) → scalar score to predict human preference. This is a classification problem: predict which response humans prefer.
  3. PPO Training: Use the reward model as a reward signal. Run RL (PPO algorithm) to update the SFT model to maximize expected reward, with a KL penalty to prevent it from drifting too far from the SFT model (prevents "reward hacking" — gaming the reward model).
PPO Objective: L(θ) = E_t[min(rₜ(θ)·Aₜ, clip(rₜ(θ), 1-ε, 1+ε)·Aₜ)] - β·KL[πθ || π_ref] rₜ(θ) = πθ(aₜ|sₜ) / π_old(aₜ|sₜ) (probability ratio) Aₜ = Rφ(response) - V(prompt) (advantage = reward minus baseline) KL term: keeps model close to SFT model (prevents reward hacking) Modern approaches also use DPO (Direct Preference Optimization) — simpler, no explicit reward model needed.

PEFT: Parameter-Efficient Fine-Tuning

Full fine-tuning of a 70B model needs ~280GB GPU memory. PEFT fine-tunes a tiny fraction of parameters:

LoRA (Low-Rank Adaptation, Hu et al. 2021)

Full fine-tune: W' = W + ΔW (ΔW ∈ ℝ^{d×k}, very large) LoRA: W' = W + BA (B ∈ ℝ^{d×r}, A ∈ ℝ^{r×k}, rank r << d or k) Typical: d=k=4096, r=16 → ΔW has 4096×4096 = 16.7M params BA has (4096×16)+(16×4096) = 131K params = 0.8% of original! Only A and B are trained. W is frozen. At inference: merge W+BA for zero overhead. 
# Using HuggingFace PEFT library
from peft import get_peft_model, LoraConfig

config = LoraConfig(
    r=16,              # rank
    lora_alpha=32,     # scaling factor
    target_modules=["q_proj", "v_proj"],  # which layers to LoRA-ify
    lora_dropout=0.05,
    task_type="CAUSAL_LM"
)
model = get_peft_model(base_model, config)
model.print_trainable_parameters()
# "trainable params: 4,194,304 || all params: 6,742,609,920 || trainable%: 0.0622"

Most Likely Technical Questions — And Your Answers

Q: Explain backpropagation. How do gradients flow through a network? ▼

Answer structure: Forward pass computes predictions and loss. Backward pass computes ∂L/∂w for every weight using the chain rule, working backward from the output. At each layer: gradient of weights = upstream gradient × this layer's input. Gradient to pass down = transposed weight matrix × upstream gradient. PyTorch's autograd builds a computation graph during forward pass and traverses it in reverse during backward(). The key insight: the chain rule allows exact gradient computation in O(n) passes through the network, not O(n²) separate passes.

Q: What is the vanishing gradient problem and how do LSTMs solve it? ▼

Answer: In plain RNNs, gradients flow back through T time steps by multiplying Wₕₕ T times. If the dominant eigenvalue of Wₕₕ < 1, gradients shrink exponentially → vanish → early time steps get near-zero gradient. LSTMs solve this via the cell state: Cₜ = fₜ⊙Cₜ₋₁ + iₜ⊙C̃ₜ. This is additive, not multiplicative. The gradient ∂Cₜ/∂Cₜ₋₁ = fₜ (the forget gate), which can be maintained close to 1.0. So gradients can flow unchanged for hundreds of steps through the cell state "highway."

Q: Explain attention. What is scaled dot-product attention and why the √dₖ scaling? ▼

Answer: Attention(Q,K,V) = softmax(QKᵀ/√dₖ)·V. Queries, keys, values are linear projections of the input. QKᵀ computes similarity scores (dot products) between queries and keys. Softmax normalizes these into weights. The weighted sum of values is the output. The √dₖ scaling: for high-dimensional vectors, dot products grow in magnitude proportional to dₖ (since each of the dₖ terms contributes independently). Without scaling, softmax saturates into one-hot distributions (near-zero gradients). Dividing by √dₖ keeps the variance of the dot products around 1, maintaining useful gradients.

Q: Batch Norm vs Layer Norm — when do you use each and why? ▼

Answer: Batch Norm normalizes across the batch dimension per feature (uses statistics from the entire mini-batch). Layer Norm normalizes across the feature dimension per sample (statistics computed per-example). BN: great for CNNs and image models where batch statistics are stable. Breaks with batch size 1 and different statistics at inference. LN: great for RNNs, LSTMs, Transformers — works with any batch size including 1, same computation at train and inference. Transformers use LN because language generation is often batch_size=1 and the sequence lengths vary, making batch statistics unreliable.

Q: Why is AdamW better than Adam + L2 regularization? ▼

Answer: In Adam+L2, weight decay is added to the gradient: g = ∇L + λw, and then the full g is scaled by the adaptive learning rate 1/√v̂. This means frequently updated parameters (with large v̂) effectively get less weight decay than infrequently updated ones — the regularization effect is corrupted by the adaptive scaling. AdamW decouples: w = (1-ηλ)w - η·m̂/(√v̂+ε). Weight decay is applied directly, independent of the gradient statistics. This gives more consistent and effective regularization, especially for Transformers where different parameter groups have very different gradient magnitudes.

ML System Design Framework

For any ML system design question, follow this structure:

  1. Problem formulation: What is the exact ML task? (classification, ranking, generation). What does success look like?
  2. Data: Where does it come from? Labels? Volume? Quality issues? Real-time vs batch?
  3. Features / Modeling: Which architecture? Why? Alternatives considered?
  4. Training: Loss function, optimizer, regularization, evaluation metrics (offline). How to avoid data leakage?
  5. Deployment: Latency requirements? Batch vs real-time serving? Model size vs accuracy tradeoff?
  6. Monitoring: How do you detect distribution shift, model degradation? Online metrics vs offline metrics gap?
  7. Iteration: How do you improve the system over time? A/B testing? Feedback loops?
